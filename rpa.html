RPA is an automated system designed to process high-volume shipment data via Excel. It streamlines ASN creation by validating your data against our warehouse requirements automatically.

1. Prepare the RPA Excel File

Ensure the following mandatory fields are correctly populated in the template:

Sequence
PO Number
Invoice & Date
Barcode / SKU
Quantity
Unit & Total Price
Currency
Log ID
Expected Weight
INCO Terms
Country Origin/Destination
Mode of Transport
Formatting Rules:
  • Sequence: The first column must be a sequence (1, 2, 3...). Drag the sequence down to match the total number of lines in your file.
  • Invoice Date: Must use the format MM/DD/YYYY 00:00:00 (e.g., 02/12/2026 00:00:00).
  • Packaging Units: Put Number of Cartons in UDF 4 and Number of Pallets in UDF 5.
  • File Naming Convention Save your completed file with the exact format:
    ASN_RPA_001_(Date)_(LOG ID)

    (Note: Increase "001" sequence based on the number of times you've pushed this file through the input folder)

  • Submission Place the file into the designated Input Folder. The system will begin processing automatically.
  • Analysis Period Wait approximately 5-10 minutes for the RPA bot to complete the data analysis.
  • Check Results
    • Validation Complete: If no errors were found, the file will move here. Your ASN is now being transmitted. You can find your transmitted ASN by visiting the respective warehouse system (WM9 or SCE11) and pasting your LOG ID into the Remarks section of the ASN tab.
    • Error File: If issues are identified, the file will move here. Open it to see the system-generated error description.
Next Step: If a file ends up in the "Error" section, you must amend the issues identified in the log, update the file, and rerun it through the Input Folder.

Note: The highlighted fields in the template are required to be filled out. As a reminder, you must fill out either the Barcode or SKU section, but not both.

For certain items, barcodes may need to be switched to SKUs due to an integration error in the barcode itself. If an SKU is used for a specific item, the barcode should be removed (and vice versa) to ensure successful processing.
